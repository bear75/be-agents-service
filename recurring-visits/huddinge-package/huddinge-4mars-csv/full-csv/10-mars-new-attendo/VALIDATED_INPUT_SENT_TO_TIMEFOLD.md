# Validated input sent to Timefold

**Yes.** The validated input sent to Timefold is **`input_81_10mars_2w.json`** in this folder. That is the only file that is POSTed as the FSR `modelInput` when you run `submit_to_timefold.py solve` or that is checked by `submit_to_timefold.py validate`.

The other files in this folder (ANALYSIS_SUMMARY_81_2w.md, analyze_csv_to_json_81_2w.txt, dependency_feasibility_81_2w.json, map_visits_time_windows_81_2w.txt, CSV_DIFF_ANALYSIS.md) are **analysis and supporting artifacts** for the same 81-client 2-week run; they are not the payload sent to Timefold.

## Where the input is stored

| File | Role |
|------|------|
| **`input_81_10mars_2w.json`** | **The validated FSR modelInput sent to Timefold.** 3746 visits, 39 vehicles, 446 shifts. Generated by `attendo_4mars_to_fsr.py` from `huddinge-81-clients - Data.csv` and `address_coordinates.json`. |

## Validations applied

### 1. Timefold validate (submit_to_timefold.py validate)

Run before solve. All must pass:

| Check | Description | Result for this input |
|-------|--------------|------------------------|
| **Shifts** | No shift with zero or negative work time (minStartTime/maxEndTime). | OK |
| **Visit time windows** | Every visit (standalone + in visitGroups) has valid time windows (start ≤ end, ISO duration, etc.). | OK |
| **FSR schema** | modelInput conforms to Timefold FSR schema (RoutePlanInput, Visit, Vehicle, VehicleShift, TimeWindow, locations, etc.). | OK |

Command used:
```bash
python3 submit_to_timefold.py validate .../input_81_10mars_2w.json
```
Output: `Validation OK (shifts, visit time windows, FSR schema).`
