# Deduplication Guide for Seed Scripts

## Overview

This guide ensures all seed scripts follow a consistent deduplication strategy to prevent duplicate data in the database.

## Core Principles

1. **Use Unique Constraints**: Always rely on schema unique constraints (slug, orgNumber, etc.)
2. **Always Use Upsert**: Never use `create()` - always use `upsert()` or check existence first
3. **Match by Canonical Identifiers**: Priority order: orgNumber > slug > name+legalName+orgType
4. **Pilot Municipalities**: Stockholm and Nacka use JSON seed as source of truth - CSV seed should skip them

## Unique Constraints in Schema

### Provider

- `slug` @unique
- `orgNumber` @unique
- `hubspotCompanyId` @unique

### Municipality

- `slug` @unique
- `municipalityCode` @unique

### ProviderMunicipality

- `@@unique([providerId, municipalityId])`

### QualityMetric

- `@@unique([presenceId, year, source])`

### ProviderFinancials

- `@@unique([providerId, year, source])`

## Provider Deduplication Strategy

### Priority Order

1. **orgNumber** (most reliable)
   - Real orgNumbers (10 digits) are canonical identifiers
   - Auto-generated orgNumbers (MUN/PRV) are less reliable but still unique

2. **slug** (fallback)
   - URL-friendly identifier
   - Should be deterministic based on name

3. **name + legalName + orgType** (last resort)
   - For municipal: match by name only
   - For private: match by name + legalName
   - Allow orgType to differ (UNKNOWN vs PRIVATE/MUNICIPAL)

### Example Usage

```typescript
// Find existing provider
const existing = await findExistingProvider(prisma, {
  orgNumber: provider.orgNumber,
  slug: provider.slug,
  name: provider.name,
  legalName: provider.legalName,
  orgType: provider.orgType,
});

if (existing) {
  // Update existing
  await prisma.provider.update({
    where: { id: existing.id },
    data: {
      /* updates */
    },
  });
} else {
  // Create new
  await prisma.provider.create({
    data: {
      /* new provider */
    },
  });
}
```

## Municipality Deduplication

### Strategy

1. Match by `slug` (primary)
2. Match by `municipalityCode` (fallback)
3. Match by `name` (last resort)

### Example Usage

```typescript
const existing = await findExistingMunicipality(prisma, {
  slug: municipality.slug,
  municipalityCode: municipality.code,
  name: municipality.name,
});

if (existing) {

      /* updates */
    },
  });
} else {
  await prisma.municipality.create({
    data: {
      /* new municipality */
    },
  });
}
```

## ProviderMunicipality Presence Deduplication

### Strategy

- Use unique constraint: `@@unique([providerId, municipalityId])`
- Always use `upsert()` with this constraint

### Example Usage

```typescript
await prisma.providerMunicipality.upsert({
  where: {
    providerId_municipalityId: {
      providerId: provider.id,
      municipalityId: municipality.id,
    },
  },
  create: {
    providerId: provider.id,
    municipalityId: municipality.id,
    status: "ACTIVE",
    // ... other fields
  },
  update: {
    status: "ACTIVE",
    // ... update fields
  },
});
```

## Pilot Municipalities

Stockholm and Nacka are "pilot municipalities" that use JSON seed data as the source of truth. CSV seed scripts should skip these municipalities.

### Check if Pilot Municipality

```typescript
import { isPilotMunicipality } from "./utils/deduplication-strategy.js";

if (isPilotMunicipality(municipalitySlug)) {
  // Skip this municipality in CSV seed
  continue;
}
```

## Seed Script Requirements

All seed scripts MUST:

1. ✅ Use `upsert()` instead of `create()` when possible
2. ✅ Check for existing entities before creating
3. ✅ Use canonical identifiers (orgNumber, slug) for matching
4. ✅ Skip pilot municipalities in CSV-based seeds
5. ✅ Handle null/undefined values gracefully
6. ✅ Use transaction batches for performance when creating many records

## Verification

After seeding, verify no duplicates:

```typescript
// Check for duplicate providers by orgNumber
const duplicates = await prisma.$queryRaw`
  SELECT "orgNumber", COUNT(*) as count
  FROM "providers"
  WHERE "orgNumber" IS NOT NULL
  GROUP BY "orgNumber"
  HAVING COUNT(*) > 1
`;

// Check for duplicate presences
const duplicatePresences = await prisma.$queryRaw`
  SELECT "providerId", "municipalityId", COUNT(*) as count
  FROM "provider_municipality_presences"
  GROUP BY "providerId", "municipalityId"
  HAVING COUNT(*) > 1
`;
```

## Troubleshooting

### Issue: Duplicate providers created

**Solution**: Ensure seed script uses `findExistingProvider()` or checks by orgNumber/slug before creating.

### Issue: Duplicate presences

**Solution**: Always use `upsert()` with `providerId_municipalityId` unique constraint.

### Issue: Pilot municipality has duplicates

**Solution**: Ensure CSV seed scripts skip pilot municipalities using `isPilotMunicipality()`.

### Issue: Auto-generated orgNumbers causing duplicates

**Solution**: Use name+legalName+orgType matching as fallback when orgNumber is auto-generated.
