Command Center & Product Owner Guide

üëî Product Owner Guide

You are the Product Manager. The orchestrator is your Scrum Master. The specialists are your Development Team. You define WHAT to build, they figure out HOW.

Your Daily Workflow

1

Evening: Define Tomorrow's Sprint Goal

Create a priority file describing what you want built. Be specific about outcomes, not implementation.

Create engineering priority
cat > ~/HomeCare/beta-appcaire/reports/priorities-$(date -v+1d +%Y-%m-%d).md <<'EOF' # Priority 1 **Description:** Add employee certifications tracking **Expected outcome:** - Database schema for certifications with expiry dates - GraphQL API for CRUD operations - UI page to view and manage certifications - Automated expiry notifications **Files:** - apps/dashboard-server/src/schema.prisma - apps/dashboard/src/pages/certifications/ **Complexity:** Medium EOF
2

Night: Team Executes Sprint (Automatic)

10:30 PM - Review agent scans yesterday's work, extracts learnings
11:00 PM - Scrum Master (orchestrator) reads your priority, creates sprint plan, coordinates specialists, creates PR

3

Morning: Review Sprint Deliverable

Check the PR created overnight. It's in draft mode waiting for your review.

Review overnight PR
gh pr list gh pr view [NUMBER] # Review code changes, test locally if needed
4

Morning: Accept or Reject Work

If the sprint met your acceptance criteria, merge. Otherwise, provide feedback in PR comments.

Accept work (merge PR)
gh pr merge [NUMBER] --squash
Or request changes (reject)
gh pr comment [NUMBER] --body "Please fix: ..."

Key Principles

Focus on Outcomes

Define WHAT you want (user value), not HOW to build it (implementation details). Let the team decide technical approach.

Trust the Team

Specialists know their domain. Backend knows database design, Frontend knows UI patterns. Review results, not process.

Compound Learning

Team learns from mistakes. Same error won't happen twice. Velocity increases over time as knowledge compounds.

Verification Gate

No broken PRs reach you. Verification specialist blocks PRs that fail type-check, build, or security audit.
